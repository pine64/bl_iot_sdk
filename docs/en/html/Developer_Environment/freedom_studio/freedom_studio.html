3. Freedom Studio¶

This document describes the use of Freedom Studio.

3.1. Import project¶

  • First start , open on the toolbar , select the option under the menu to import the project.Freedom StudioFile > importGeneralExisting Projects into Workspace

3.2. Debug¶

  • Let the board up and running, use shortcut keys F11to start Debug, Freedom Studio will compile the project (note confirming the use of keyboard shortcuts when using the .launch file name is bl_iot_sdk_debug_freedom_studio.launch, can be viewed via the toolbar Run> Debug As>)

  • Use shortcuts F8resume look, you can see the program stopped at the main function, you can now click below the 3 button. The meanings arevoid bfl_main()

    • The first Step Into (F5) is executed in a single step, and when it encounters a sub-function, it enters and continues to execute in a single step;

    • The second Step Over (F6) will not enter the sub-function for single-step execution when it encounters the sub-function in the function when single-stepping. Instead, the sub-function is executed and stopped, that is, the whole sub-function is step;

    • When the third Step Return (F7) is stepped into the sub-function, you can use Step Return to execute the rest of the sub-function and return to the previous function.

  • May be in c代码the far left of the orange bar window, double-click increase, remove the breakpoint.

  • You can also see through the right side Disassemblyto see the corresponding compilation window (the window if you can not through the top of the toolbar to add).Window > show view > Disassembly
