2. helper¶

This section mainly introduces some commands currently supported, including system commands and user commands. Of course, some command boards may not support them. You can helpcheck the supported commands. For more commands, please refer to the source code or add them by yourself

2.2. Common commands¶

  • View all supported commands

    You can TABview it by pressing the key , or enter the command helpas follows:

    # help

  • Configure the temporary ssid and passwd of wifi (restart to restore the default value)

    For example, configure wifi: bl_wifi_005and password: 123456789 enter the following command from the command line:

    # wifi_sta_connect bl_wifi_005 123456789

  • Configure the default ssid and passwd of wifi

    For example, configure wifi: bl_wifi_005and password: 123456789 enter the following command from the command line:

    # psm_set conf_ap_ssid bl_wifi_005

    # psm_set conf_ap_psk 123456789

  • Start AP

    Use wifi_ap_startcommands to start the AP, but you need to psm_eraseerase the STA related configuration information and restart before use . The activated AP SSID is BL60X_uAP_a HEX string with the last 3 bytes of the MAC address and the password isbouffalolab

  • System restart

    # reboot

  • Broadcasting and recording

    This module requires hardware with SD card support, temporarily only supports pcm format files, other formats are to be updated

    • Configure audio sampling frequency

      # audio_config 11p025

      If there is no parameter, 16Khz is used by default, and 8k, 16k, 24k, 32k, 48k, 11p025, 22p05, 44p1 sampling frequencies are supported (11.025khz is represented by 11p025, others are similar), for example, the sampling frequency is set to 11.025Khz here

    • recording

      # audio_record test.pcm

      If there is no parameter, the default recording is placed in the root directory of the sd card, the recording file name is record.pcm, where the configuration file name test.pcm is stored in the root directory test.pcm file, and the default recording is about 10S

    • Sound release

      # audio_play test.pcm

      If there is no parameter, the default is to use the recording file name record.pcm in the root directory of the SD card, where the file name test.pcm is configured, that is, the test.pcm audio in the SD card is played

  • Picture Transmission

    Use mjpeg_startcommands to start image transmission related tasks

  • Save picture to SD card

    This module requires hardware with SD card support, temporarily only supports jpeg format files, other formats are to be updated

    • Configure the quality of mjpeg images

      # mjpegsd_config 50

      Temporarily support 6 types of quality 5, 10, 25, 50, 75, 100

    • Start saving picture

      # mjpegsd_start

      Start saving picture, save 50 frames of data by default

    • note

      • As above, the example command will save 50 0.jpeg, 1.jpeg...49.jpeg pictures in the root directory of the SD card

  • Save AVI to SD card

    This module requires hardware with SD card support, temporarily only supports AIV format files, other formats are to be updated

    • Configure the quality of mjpeg images

      # avisd_config 50 bl_avi_q50.avi

      Temporarily support 6 types of quality 5, 10, 25, 50, 75, 100

    • Start save avi

      # avisd_start

      Start saving avi, save 750 frames by default

    • note

      • As above, the sample command will save a "bl_avi_q50.avi" file in the root directory of the SD card

      • The memory card is recommended to be formatted as the allocation unit 64KB, and the file system is FAT32

      • After avisd_start, if you want to modify the quality, you need to restart the development board

      • The module needs to be supported by psram and camera, and needs to enable PSRAM and camera related switches

  • QR code distribution network

    This module connects to WiFi through the QR code of the mobile phone sharing hotspot

    • Start command

      qrcode_connect_wifi

      After entering the command, the camera will be initialized and then start to recognize the QR code in a loop

    • note

      • After the QR code is successfully recognized, the QR code information will be printed first, if it is not the QR code for sharing hotspots, it will continue to scan

      • Supports SSID as Chinese, but requires unicode encoding rule to be UTF-8

      • Support open hotspot

      • This module needs psram and camera support, and needs to enable PSRAM and camera related switches
