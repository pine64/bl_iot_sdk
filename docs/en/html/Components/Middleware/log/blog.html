8. blog¶

8.1. Overview¶

Blog is an ultra-lightweight log component, very suitable for resource-sensitive software projects.

8.2. Function introduction¶

Blog simplifies log into 3 categories, namely components, files, and private logs, and supports software dynamic modification and macro shutdown solutions (not occupying rom). Hereinafter, the dynamic modification is collectively referred to as a soft switch, and the macro complete shutdown scheme is a static switch.

  • The component log manages the log with the highest priority. The following file logs and private logs are restricted by this switch

  • File log management In the log priority, the following private logs are restricted by this switch, and the file log is managed by the component log.

  • Private log management This log has the lowest priority and is managed by component log and file log

8.3. Log level description¶

The levels are as follows, where all is the lowest, that is, all logs are output

all : 所有log均输出，其实等同于all
debug : debug及以上
info : info及以上
warn : warn及以上
error : error及以上
assert : assert及以上
never: 所有log均不输出，其实等同于assert

8.4. How to use¶

Contains the necessary header file and component are provided log, file log, private log.#include <blog.h>

#include <blog.h>
BLOG_DECLARE(blog_testc2);

void func(void)
{
    blog_debug("blog_testc2 debug\r\n");
    blog_info("blog_testc2 info\r\n");
    blog_warn("blog_testc2 warn\r\n");
    blog_error("blog_testc2 error\r\n");
    blog_assert("blog_testc2 assert\r\n");

    blog_debug_user(blog_testc2,"blog_testc2 debug user\r\n");
    blog_info_user(blog_testc2,"blog_testc2 info user\r\n");
    blog_warn_user(blog_testc2,"blog_testc2 warn user\r\n");
    blog_error_user(blog_testc2,"blog_testc2 error user\r\n");
    blog_assert_user(blog_testc2,"blog_testc2 assert user\r\n");
}

  • Component log switch

    • The static switch is in the corresponding proj_config.mk file directory, and the corresponding component name is added to the LOG_ENABLED_COMPONENTS configuration. For example, here you need to add the blog_testa blog_testb blog_testc component static switch, and other components are closed by default LOG_ENABLED_COMPONENTS:=blog_testa blog_testb blog_testc

    • The software switch enables the log output level by entering the following commands, such as logset level component_name: blogset assert blog_testc

  • File log management

    • Add this line of code to the corresponding *.c file for the static switch. Note that whether it is on or off, you must choose one. 默认就是开 #define BLOG_HARD_DECLARE_DISABLE 1    // 关

    • The software switch enables the log output level by entering the following command, such as logset level component_name.file_name for example: blogset info blog_testc.blog_testc2_full

  • Private log management

    • Add BLOG_DECLARE(...) if you use static switch instead of adding this line directly. BLOG_DECLARE(blog_testc2);    // 打开，其中　"blog_testc2"为用户自定义

    • The software switch enables the log output level by entering the following command, such as logset level component_name.file_name.pri_name for example: blogset debug blog_testc.blog_testc2_full.blog_testc2
