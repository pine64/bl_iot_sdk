4. UART_echo¶

4.1. Overview¶

This example mainly introduces the process of returning the received data to the sender through UART1.

4.2. Preparation and use steps¶

  • Before you need a USB转TTLserial line, and disposed SDK directory bl_iot_sdk/tools/flash_tool/bl602/device_tree/bl_factory_params_evb_40M.dtsfile uart, the specific configuration of the present embodiment can refer to.

uart {
    #address-cells = <1>;
    #size-cells = <1>;
    uart@4000A000 {
        status = "okay";
        id = <0>;
        compatible = "bl602_uart";
        path = "/dev/ttyS0";
        baudrate = <2000000>;
        pin {
            rx = <7>;
            tx = <16>;
        };
        feature {
            tx = "okay";
            rx = "okay";
            cts = "disable";
            rts = "disable";
        };
    };
    uart@4000A100 {
        status = "okay";
        id = <1>;
        compatible = "bl602_uart";
        path = "/dev/ttyS1";
        baudrate = <115200>;
        pin {
            rx = <3>;
            tx = <4>;
        };
        feature {
            tx = "okay";
            rx = "okay";
            cts = "disable";
            rts = "disable";
        };
    };
};

  • Steps for usage:

  • The board gpio3and gpio4and GNDrespectively USB转TTLthe serial lines TXD, RXD, GNDare respectively connected;

  • Compile customer_app/sdk_app_uart_echothe project and download the project;

  • Open a serial terminal window A (115200 baud rate, used to receive and send uart messages), and open another serial terminal window B (baud rate 2000000, used to print log). The power board can see the window A received 1234567890abcdefg, the print log B is stopped at the window , enter in the window A (no echo) can be seen in the port B and the message, this time successful demonstration described.send case1234567890abcdefgrecv caseend

4.3. Application examples¶

  • Call aos_write()interface to send data to the terminal via UART1, and waits for the terminal returns the received data, data same as transmission and reception, the log will print the message content UART1 port sent and received.

aos_write(fd, send_recv_log, strlen(send_recv_log));
log_step(ci_table_step_send);

while (1) {
    length = aos_read(fd, buf_recv, strlen(send_recv_log));
    if (length != strlen(send_recv_log)) {
        continue;
    }
    if (memcmp(buf_recv, send_recv_log, strlen(send_recv_log)) == 0) {
        printf("recvbuff:%s\r\n", send_recv_log);
        log_step(ci_table_step_recv);
        break;
    }
    vTaskDelay(10);
}
log_step(ci_table_step_end);
